# Internationalization

## Hugo model

`hugo.yaml` defines two native Hugo languages. `zh-cn` is default and remains at root; `en` uses `/en/`. The content directories are:

```text
content/
├── zh/
│   ├── projects/
│   ├── research/
│   ├── open-source/
│   ├── notes/
│   └── about/
└── en/
    └── (same relative paths)
```

Paired files use the same relative path and `translationKey`. Hugo then emits language-specific canonicals, RSS, sitemap entries, Open Graph locale, and `hreflang`; the project head hook also emits `x-default`.

## Language switch behavior

The Header checks `.Translations` for the current page. If a translation exists, the switch goes directly to it. If not, it falls back to the target-language home and exposes the fallback in its title. Theme choice is stored independently by PaperMod in `localStorage`, so switching language does not reset light/dark mode.

Search uses `relLangURL`: `/search/` for Chinese and `/en/search/` for English. Giscus also receives the correct UI language.

## Editorial rules

- Chinese is canonical; update it first.
- Do not hand-copy a new English file without translation metadata.
- Set `translation.reviewed: true` only after a human review.
- Set top-level `translation_locked: true` to protect a hand-maintained English page.
- Use `translate: false` on Chinese content that should have no machine translation.
- Keep legacy Chinese URLs through `aliases`; do not create unconditional language redirects.

Run `python scripts/validate_content.py` to detect missing pairs, mismatched keys, stale hashes, invalid statuses, and broken internal Markdown links.
